# Code Review Standards

这些文件是 `project-review` 内部 `code-review` lane 的参考标准库。

当前已吸收的重点模块：

- `performance.md`
- `batch_robustness.md`
- `api_status_codes.md`
- `api_response_format.md`
- `api_error_handling.md`
- `testability.md`
- `code_style.md`

使用原则：

- 规则脚本先收集候选风险
- reviewer 再结合这些标准和仓库证据判断风险是否成立
- 不把标准库本身当成“自动判定真相”的替代品
