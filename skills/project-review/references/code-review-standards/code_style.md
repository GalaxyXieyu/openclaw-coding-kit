# 代码规范与架构一致性（P2）

## 5. 命名规范与一致性
- [ ] 变量、函数、类使用统一命名风格（camelCase 或 PascalCase）
- [ ] 文件名遵循项目规范，避免冗余命名（如 `service/data_service.py` 应简化为 `service/data.py`）
- [ ] 命名语义清晰，易于区分

## 6. 导入导出清理
- [ ] 是否存在未使用的 import 语句？
- [ ] 是否有循环依赖和重复导入？
- [ ] 导入路径是否清晰、符合项目结构？

## 7. 架构一致性
- [ ] 本次变更是否与现有代码架构风格一致？
- [ ] 新增功能是否遵循项目既定的架构模式？
- [ ] 是否引入了不必要的架构差异？

## 8. 数据检索方法一致性
- [ ] 数据库查询、API 调用是否与现有模式一致？
- [ ] 是否在同一项目中混用不同的 ORM 用法？

## 9. 注释与文档风格
- [ ] 注释和文档字符串风格是否统一？
- [ ] 关键逻辑是否有注释说明？
- [ ] 是否存在过度注释或无意义注释？
